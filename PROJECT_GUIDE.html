01 — Project Overview

WHAT IS THIS APP?

An interactive Oracle Exadata X8M Lab Simulator built with Streamlit and OpenAI. It provides a structured DBA/DMA curriculum, AI-powered lesson walkthroughs, and a hands-on command playground — all in a browser, no real Exadata hardware needed.

CURRICULUM

9 modules (M0–M8), 20+ lessons covering CellCLI, DCLI, ASM, RMAN, SRVCTL, Data Guard, Smart Scan, Incident Response.

AI TRAINER

ExaSimBot (OpenAI GPT-4o) simulates demos, evaluates practice commands, runs scenarios and quizzes.

PLAYGROUND

50 hands-on exercises across 8 categories. Type real commands, get AI feedback, track score per category.

06 — Git Pull (Get Latest Code)

BEFORE YOU START WORKING — ALWAYS PULL FIRST
# 1. Navigate to the project directory
cd /home/cmidbmod/exa/exadatasimulator

# 2. Check current status
git status

# 3. Pull latest changes from GitHub
git pull origin main

# 4. Confirm you are up to date
git log --oneline -5
⚠ Never skip the pull step. If someone else (or Streamlit AI) edited the repo on GitHub and you edit locally without pulling first, you will get a merge conflict when you try to push.

What to do if pull says "merge conflict"

# See which files conflict
git status

# Option A — Keep your local version (discard GitHub changes)
git checkout --ours <filename>
git add <filename>
git commit -m "resolve conflict — kept local version"

# Option B — Take GitHub version (discard your local changes)
git checkout --theirs <filename>
git add <filename>
git commit -m "resolve conflict — took remote version"

# Option C — Nuclear: throw away ALL local changes, match GitHub exactly
git fetch origin
git reset --hard origin/main

07 — Git Commit & Push (Save Changes)

STANDARD PUSH WORKFLOW — DO THIS EVERY TIME YOU MAKE CHANGES
# 1. Go to project directory
cd /home/cmidbmod/exa/exadatasimulator

# 2. See what changed
git status
git diff                      # show line-by-line changes

# 3. Stage the files you changed
git add app.py                 # specific file
git add pages/2_Playground.py  # another file
git add .                      # OR: stage everything (use carefully)

# 4. Commit with a clear message
git commit -m "Add feature: describe what you changed"

# 5. Push to GitHub
git push https://khadayatepa:YOUR_TOKEN@github.com/khadayatepa/exadatasimulator.git main
Tip — store your token in the remote URL once so you never need to type it again: 
git remote set-url origin https://khadayatepa:YOUR_TOKEN@github.com/khadayatepa/exadatasimulator.git
# Now you can just run:
git push origin main

GitHub Personal Access Token (PAT)

  1. Go to GitHub Settings
    github.com → top-right avatar → Settings
  2. Developer Settings → Personal Access Tokens
    Left sidebar → Developer Settings → Personal Access Tokens → Tokens (classic)
  3. Generate new token
    Click "Generate new token (classic)" → tick the repo checkbox → set expiry → Generate
  4. Copy the token immediately
    Starts with ghp_... — you only see it once. Store it safely (password manager).
  5. Revoke old tokens
    Any token shared in chat / email / Slack must be revoked immediately from the same Tokens page.

10 — Running Locally

  1. Clone the repository
    git clone https://github.com/khadayatepa/exadatasimulator.git
cd exadatasimulator
  2. Install dependencies
    pip install -r requirements.txt
  3. Run the app
    streamlit run app.py
# Opens at http://localhost:8501
  4. Enter your OpenAI API key
    Type your sk-... key in the API Key field on the Home page config bar.
  5. Optional — use a secrets file
    # Create .streamlit/secrets.toml
mkdir .streamlit
# Add: openai_key = "sk-..."

11 — Environment & Secrets

⚠ Never commit API keys or tokens to git. The .gitignore should include .streamlit/secrets.toml and any .env files.
OPENAI API KEY

3 ways to provide it (in priority order):

# 1. Via Streamlit secrets (recommended for cloud)
# .streamlit/secrets.toml:
openai_key = "sk-..."

# 2. UI input on the Home page
# (works anywhere, stored in session state only)

# 3. Environment variable (local dev)
export OPENAI_API_KEY=sk-...
GITHUB PAT TOKEN

Used only for git push — never stored in the app:

# Store in remote URL (one-time setup)
git remote set-url origin \

# Then push is simply:
git push origin main

# github.com → Settings → Developer Settings → PAT

12 — Troubleshooting

ProblemCauseFix
Streamlit Cloud shows old code Push didn't reach GitHub Run git log --oneline -3 — if local is ahead of origin, the push failed. Re-push with token.
git push → 403 Forbidden Token lacks repo write permission Generate new classic PAT with repo scope checked.
git push → "could not read Username" No credentials configured Use full URL with token: git push https://user:token@github.com/...
App shows blank/white page Python import error Streamlit Cloud → Logs tab. Usually a missing import or syntax error.
Playground page 404 pages/2_Playground.py not in repo Confirm file was committed: git show HEAD --stat
OpenAI error in Demo/Practice Invalid or missing API key Check key in Home config bar. Verify key at platform.openai.com → API Keys.
ModuleNotFoundError: openai Dependency not installed Check requirements.txt has openai>=1.30.0. Run pip install -r requirements.txt.
Merge conflict on pull Local and remote both changed same file See Section 06 for resolution steps.
Session state resets on every page switch Normal Streamlit behaviour — expected init_state() uses if k not in st.session_state to preserve values across reruns.
